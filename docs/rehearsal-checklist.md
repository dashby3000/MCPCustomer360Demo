# Rehearsal Checklist

Use this before a live demo.

## Setup

- Run `npm run smoke`
- Keep [demo-playbook.md](/Users/dashby/Workarea/AmaroDemoData/docs/demo-playbook.md) open
- Keep [golden-answers.md](/Users/dashby/Workarea/AmaroDemoData/docs/golden-answers.md) open
- Keep [demo-questions.txt](/Users/dashby/Workarea/AmaroDemoData/docs/demo-questions.txt) ready to paste
- Make sure `customer360` is enabled in the MCP client along with the source MCPs

## Best live sequence

1. Ask: `Which accounts are at risk for churn, and why?`
2. Drill into: `Use customer360 first. Show me everything going on with Redwood Bank.`
3. Contrast with: `Use customer360 first. Now show me what Alturas Health is doing differently.`
4. Ask: `What do my best reps, CSMs, and support employees do differently from everyone else?`
5. Finish with: `What features do customers complain about or request the most? Build a roadmap for me.`

## What good answers should feel like

- Cross-source, not single-source
- Explanatory, not just list-based
- Specific about accounts, people, and themes
- Nuanced enough to distinguish noisy accounts from true churn risk
- Clear about what is a churn blocker versus an expansion lever

## If the answer is weak

- Ask it to cite more than one system
- Ask it to use `customer360` first for grounded account facts
- Ask for evidence behind the conclusion
- Ask it to compare a risky account with a strong one
- Ask it to separate support volume from actual churn risk
- Ask it to distinguish feature complaints from strategic roadmap opportunities
